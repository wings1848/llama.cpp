技术设计第 10.2 节的 TPS 数据与测试报告第 4 节完全不符（见上文 1.2 节表格）。两者都声称包含 B0/B1/B2 修复。如此大的差异会直接误导读者对 SWLP 效益的判断。

### 2.3 优化方案状态矛盾（测试报告 vs 优化文档）

| 状态项 | 测试报告 | 优化文档 |

这种矛盾会让维护者和读者都感到困惑。

> **修复后状态**: 所有 SWLP CLI 参数已统一为 `--swlp-` 前缀命名（`--swlp-window`, `--swlp-prefetch`, `--swlp-adaptive`, `--swlp-pinned-copy` 等），bench 工具、Python 测试框架与主 CLI 已完全一致。
